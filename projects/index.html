c and shell

http://gitlander.com

There can be only one gitlord, but anyone can be a gitlander. view the readme for instructions on the alpha cli client which lets you setup remote repos that you can clone and push too. Uses git shell and existing kernel security groups to provide a usuable git experience. While git access is private, your repos are mirrored and available over http.

golang + react

http://bookmarks.gitlander.com

Applying some lessons learned building a custom Meteor collection, I implemented a similiar reactive collection in go. This reactive collection backs a live feed which helps you keep track of the things you want to read. You can also search other users and look for people with a collection in an areas you're interested in. Subscribe to them, and get access to their bookmarks in your feed.

https://github.com/slofurno/minesweepers A more competetive version of minesweeper, where instead of racing against the clock you compete against other people to flag more mines. Built to test out Mithril, an alternative to React, but with much better performance. Uses csharp on the server, relying only on the standard library. Features my own minimal implementation of the websocket protocol and some non idiomatic csharp, inspired by go.

Created with react and go, TempestTrackers was a web application to facilitate the crowd sourcing of game event information in arpg path of exile. Exposes a rest api and realtime stream using the eventsource protocol, similiar in its design to firebase.

csharp

https://github.com/slofurno/fsharp-playground An experiment combining my interest in functional programming and the beta version of mvc6. Inspired by the golang playground, this provides a convenient place to write, execute, save, and share fsharp code. Running on the coreclr, microsoft's open source and cross platform execution enviroment, it is equally capable of being deployed on windows and linux.

Any time I work on an app exposing encoded strings, i reach for a library that provides slightly less compact but much more readable 32 bit encoding following crockford's spec. crocksharp is my open source csharp implementation, available on nuget and compiled to work on both the current dotnet, and the beta of the coreclr.

Because my preferred json encoding library, jil, has not been compiled against the coreclr, I've been working on a bare-knuckle library which does. Also available on nuget, but currently only supports json encoding.
